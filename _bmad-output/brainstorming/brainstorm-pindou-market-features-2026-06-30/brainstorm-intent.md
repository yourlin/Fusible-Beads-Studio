# Pindou-Studio 差异化方向 Intent

## 1. 主题与目标

**主题**：盘点拼豆 / 像素艺术类工具的市场功能版图，为 pindou-studio 找到可「以小博大」的差异化切入点。

**目标**：在穷举同类工具功能的基础上，标出 pindou-studio 尚未做、且贴合自身核心能力（CIEDE2000 颜色匹配）的差异化机会，收敛出可立项的方向。

## 2. 市场功能版图（三层结构）

会话将约 130 条功能想法收敛为三层：

- **① 桌面级基线层**（人人都有，非差异化）：导入（本地/拍照/URL/AI 生图）、图像转珠（自动颜色匹配、网格分辨率、抖动、减色）、编辑（画笔/形状/图层/对称/取色/撤销）、物料统计、导出（PNG/PDF/分块打印/色号图/工程文件）。**pindou-studio 已覆盖大部分。**
- **② 生态层**（规模化护城河，但烧钱、靠网络效应）：社区图库、图纸上传/交易市场、创作者分成、教程关卡化、搭建跟随（逐格高亮/进度打勾/陪伴模式）。
- **③ 差异化突破层**（市面普遍空白，pindou 的机会区）：库存驱动设计、实物优先（拍照数字化 + 纠错）、多媒介换算、极简风格化、AR 对位/纠错。

## 3. pindou-studio 已覆盖 vs 缺口

- **已覆盖**：图片转拼豆、CIEDE2000 颜色匹配、网格编辑、PNG/PDF 导出 —— 即基线层主体。
- **缺口（差异化层，均未做）**：库存驱动、实拍数字化与纠错、极简风格化生成、多媒介换算、AR。

## 4. 选定的 3 个差异化方向（核心）

### 方向一：库存驱动设计（我有什么珠 → 能做什么图）
- **是什么**：用户先录入手头实际拥有的珠子颜色与数量，工具反推「在这套库存约束下，这张图该怎么转 / 能做哪些图案」，并给出缺色替代建议。
- **反转的默认假设**：「先有图，再转珠」→ 改为「先有料，再推图」。
- **为何贴合 CIEDE2000**：颜色匹配引擎本就在做「目标色 → 最近色板色」的映射；这里只是把「色板」从理想全色板换成「用户真实库存子集」，并在缺色时用感知色差找最优替代 —— 同一引擎换输入。
- **初步价值假设**：消除「设计完才发现没料 / 缺色」的挫败，降低耗材浪费，提高从设计到完工的转化率。

### 方向二：实拍数字化 + 摆放纠错（实物优先）
- **是什么**：拍下随手摆好的（或半成品）拼豆成品，自动识别成可编辑的数字图纸（反向 OCR）；摆放过程中拍照检测摆错 / 漏摆的格子并提示纠正。
- **反转的默认假设**：「在屏幕上设计」「导出即结束」→ 改为「实物优先、摆放中实时纠错」。
- **为何贴合 CIEDE2000**：识别照片中每颗珠子的颜色、归一化到品牌色号、判断「实拍色 vs 应有色」是否一致，本质都是感知色差比对 —— 正是颜色匹配引擎的强项（在光照噪声下用 CIEDE2000 容差判定）。
- **初步价值假设**：打通线下实物与线上图纸的双向闭环，覆盖「存档既有作品」「质检 / 纠错」两个全新场景。

### 方向三：极简低难度风格化（极简珠图生成器）
- **是什么**：不追求最大还原原图，而是主动风格化 —— 用最少颜色、最低摆放难度生成「也好看」的珠图，并能按目标难度 / 色数分级。
- **反转的默认假设**：「追求最还原原图」「颜色越多越好」→ 改为「主动减色、低难度优先」。
- **为何贴合 CIEDE2000**：减色 / 合并相近色时，用感知色差决定「哪些颜色可安全合并而视觉损失最小」，比简单 RGB 距离更准 —— 用同一引擎做「智能简化」而非「精确匹配」。
- **初步价值假设**：降低新手 / 儿童的上手门槛与材料成本，扩大可服务人群，与可达性（色盲友好、高对比限色）天然协同。

## 5. 核心洞察

- **三个方向是同一条护城河的三个面**：本质都是把 CIEDE2000 颜色匹配引擎「反着用」——
  - 库存驱动 = 约束色板下的匹配；
  - 实拍纠错 = 反向识别 + 容差比对；
  - 极简风格化 = 感知最优的减色合并。
  一套核心能力衍生三种产品形态，复用度高、技术风险集中可控。
- **战略取舍**：应避开生态层（社区 / 图纸交易市场）那条靠规模与网络效应的烧钱赛道，集中资源打差异化突破层，发挥「轻量、技术驱动、以小博大」的优势。
