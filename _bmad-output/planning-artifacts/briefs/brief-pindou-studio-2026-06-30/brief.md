# Product Brief: Fusible Beads Studio 差异化方向

> **目的**：评估三个差异化方向（库存驱动设计 / 实拍数字化+纠错 / 极简风格化），给出独立判断与首发方向排序推荐，作为下游 PRD 立项依据。
>
> **快速路径产物**：凡标 `[ASSUMPTION]` 处为基于代码现状与头脑风暴的推断，请在 review 时直接修正。
>
> **rev.2 更新（2026-06-30）**：经市场研究 + App Store 竞品截图核实，**修正了「库存驱动是市场空白」的原始假设**——该方向已有成熟竞品（BeadMaster、Fuse Beads Studio 等）。库存驱动的差异化叙事据此从「填空白」改为「做更深」。详见各节及末尾「市场研究发现」。

## Executive Summary

**结论先行**：pindou-studio 的「图片转拼豆图纸」基线层已相当完整，但基线层是红海，不构成差异化。本简报评估头脑风暴收敛出的三个差异化方向，**分析推荐从「库存驱动设计」起步**（首发未锁定，见末尾说明）。

关键杠杆：三个方向本质上都把同一个核心资产——CIEDE2000 感知色差引擎（`shared/color`，区别于廉价 RGB 竞品的技术底座）——**反着用**。库存驱动 =「在约束色板下匹配」，实拍纠错 =「反向识别+容差比对」，极简风格化 =「感知最优的减色合并」。一套引擎衍生三种形态，复用度高，技术风险集中可控；战略上避开靠规模与网络效应烧钱的生态层（社区图库、交易市场）。

**关于差异化立论（rev.2 校正）**：市场研究显示，这三个方向并非都是「市场空白」——库存驱动已有成熟竞品，实拍纠错才是真空白。因此 pindou-studio 的差异化不能一概靠「第一个做」，而要分方向论证：库存驱动靠**匹配深度**（CIEDE2000 缺色替代）+ **Web 端形态**，实拍纠错靠**技术壁垒**，极简风格化靠**人群拓展**。

## Background：今天的 pindou-studio

- **形态**：纯浏览器、零后端的单页 Web 应用，计算全在本地完成，无网络请求/账号/后端（GitHub Pages 静态托管，另有可选 AWS 路径）。
- **已覆盖能力（基线层）**：多格式导入（含 SVG 本地栅格化）→ CIEDE2000 逐像素匹配 → 可选 Floyd–Steinberg 抖动 → 去背 → 交互式编辑器（画笔/橡皮/油漆桶/取色/撤销重做）→ 物料统计 → PNG / 带色号 PDF 导出。支持 Perler / Hama / Artkal 等色板。
- **现状定位** `[ASSUMPTION]`：个人业余维护的开源项目，目标用户为**拼豆爱好者个人玩家**，规模以自用 + 早期开源用户为主，尚未规模化推广。
- **竞品现状（rev.2，App Store 截图核实）**：拼豆 App 生态比简报初稿设想的成熟。已确证两款重度竞品：**BeadMaster**（库存盘点 + 补货助手 + 「实时查询图纸可拼状态」+ 作品集 + AI 配色）与 **Fuse Beads Studio**（22 万颗级库存管理 + 图纸转换 + 图纸库分享 + 跟拼视图，其「AI 一键识别」经核实为**电商订单截图 OCR**，非拍实物）。二者均为**移动 App、面向国内市场（带 ICP 备案）**，且都把库存做得很深。

## The Problem：基线之外，真正的痛在哪

「图片转图纸」只解决了创作链条的中间一段；真实挫败发生在两端，共性都不是「转得准不准」，而是「转出来后能否真正落地完工」——这恰恰是基线层工具集体忽略的地方：

- **开工前——料不对**：屏幕上设计得满意，真要摆时才发现手头没有这些色 / 某几色不够，只能停工补货（打断心流）或肉眼硬凑替代色（偏色）。设计稿与库存脱节，是「设计完成→成品完工」之间最高的一道坎。*（rev.2：竞品 App 已用「库存盘点 + 补货清单」缓解此痛，但解法是「叫你买」；「肉眼硬凑偏色」这半个痛点——即缺色时如何科学替代——仍无人解，正是 pindou 的切入口。）*
- **过程与收尾——线下线上断裂**：摆错/漏摆一格，往往要整片返工时才发现；做完的实物想存档成数字图纸却**没有回路**，只能重新手画。
- **门槛——对新手/儿童太重**：「最大还原」意味着色多、相邻色难分、难摆、耗材贵；新手与儿童需要的是**少色、易摆、也好看**。

## The Three Directions：候选方向评估

三个方向是同一引擎的三个面，各自反转了一个默认假设：

- **① 库存驱动设计（先有料 → 反推能做什么图）**：反转「先有图再转珠」→「先有料再推图」。用户先录入手头珠子的颜色与数量，工具在「真实库存子集」这一约束色板下转换，缺色时用 CIEDE2000 给出最优替代色与缺口提示——**同一引擎，只换输入**，技术增量最小。价值：消除「设计完才发现没料」，减少耗材浪费，提高「设计→完工」转化率。
  - **竞品现实（rev.2）**：**此方向不是空白**——BeadMaster、Fuse Beads Studio 等已做库存盘点、「图纸可拼状态」、补货清单。但核实发现一个**幸存差异化点**：现有竞品缺色时的动作是「**叫你补货**」（导出采购清单），**没有人**做「缺色时用 CIEDE2000 在你手头已有的色里找感知最优替代」。**别人说"去买"，pindou 说"用手头的替"**——这个「替代而非补货」的语义，叠加 **Web 端零安装、隐私本地、跨平台**（竞品均为国内移动 App），是仍然成立的差异化。
  - **风险**：差异化窗口比初稿设想的窄——不再靠「第一个做」，而靠匹配质量做得更深；需确认竞品确实未做感知替代（已据截图初步确认：它们只做补货）。录入成本仍是体验风险。技术风险低。
- **② 实拍数字化 + 摆放纠错（实物优先）**：反转「屏幕设计、导出即结束」→「实物优先、摆放中实时纠错」。拍照把成品/半成品识别成可编辑数字图纸（反向 OCR），并检测摆错/漏摆的格子——本质是在光照噪声下用 CIEDE2000 容差做比对。价值：打通线下↔线上双向闭环，开辟「作品存档」与「质检/纠错」两个全新场景。
  - **竞品现实（rev.2）**：**确证为真空白**——核实多款 App，所谓「AI 识别」最深的也只是**订单截图 OCR**（Fuse Beads Studio），无一做「拍物理珠子→识别成数字图纸」或摆放纠错。相邻品类有强可行性背书：乐高 **Brickit** 已工程化「拍散砖堆→CV 识别每块→告诉你能拼什么 + 位置」。
  - **风险**：**技术风险最高**。珠子的**定位/分割是 CV 难题**（颜色识别只是其后一步），光照/透视/阴影都干扰；拼豆比乐高更难（珠小、同色密集）。纯前端能否扛住识别精度是核心未知，与「零后端」约束最可能冲突 `[ASSUMPTION]`。
- **③ 极简低难度风格化（极简珠图生成器）**：反转「最大还原、颜色越多越好」→「主动减色、低难度优先」。用最少色 / 最低难度生成「也好看」的珠图，并按难度分级——用感知色差决定「哪些色可以安全合并而视觉损失最小」，是「智能简化」而非「精确匹配」。价值：降低新手/儿童门槛与材料成本，与可达性（色盲友好、高对比限色）天然协同。风险：「也好看」是**主观审美目标，难量化、难验收**，减色容易做出「能跑但成品丑」的结果，价值高度依赖输出质量与调参，迭代周期可能较长 `[ASSUMPTION]`。

### 评估小结与推荐（rev.2，含竞品空白度）

| 方向 | 引擎贴合度 | 技术风险 | 竞品空白度（市场研究核实） | 与零后端约束 | 差异化立论 |
|---|---|---|---|---|---|
| ① 库存驱动 | 最高（换输入即可） | 低 | **低**（BeadMaster 等已做得很深） | 完全兼容 | 靠**深度**：CIEDE2000 缺色替代 + Web 形态 |
| ② 实拍纠错 | 中（需新增 CV 定位） | 高 | **高**（确证真空白 + Brickit 类比） | 可能冲突 | 靠**壁垒**：CV 识别难，先发吃护城河 |
| ③ 极简风格化 | 高 | 中 | 中（基本空白，有弱风格化替代） | 完全兼容 | 靠**人群**：降门槛扩新手/儿童/色盲 |

**关键校正**：「最空白」与「最该首发」不是一回事——最空白的实拍纠错风险最高、最不空白的库存驱动风险最低，没有「既空白又低风险」的免费午餐。

**分析推荐（结论，非锁定决策）：① 库存驱动仍是性价比最高的首发**——需求已被竞品验证（不再是「赌它有人要」），风险最低、引擎复用最深，且**保有「感知替代 + Web 形态」的差异化立论**。**建议排序**：① 库存驱动（差异化深耕）→ ③ 极简风格化（低风险第二增量，与库存「约束色板」协同：库存少色 = 天然极简场景）→ ② 实拍纠错（高风险高回报、唯一真空白的远期护城河探索，需先回答「纯前端 CV 能否扛」）。

> **首发方向已锁定（2026-06-30，用户决定）：库存驱动设计**。差异化主轴 = CIEDE2000 缺色感知替代（「替代而非补货」）+ Web 端零安装形态。已进入 `bmad-prd` 立项。

## Who This Serves

- **主要用户** `[ASSUMPTION]`：拼豆/像素艺术的个人爱好者——有一定动手意愿，会为「设计完却没法落地」而真实受挫。对库存驱动方向，他们的画像是**已经攒了一批珠子、希望物尽其用、不想为单张图重复补货的人**。
- **次要用户** `[ASSUMPTION]`：新手与亲子/儿童场景（极简风格化方向的核心受众）；手作卖家与教育场景为潜在外延，**本期不作为目标**。

## Vision

差异化层走通后，pindou-studio 将从「又一个图片转珠工具」变成**贴着「从手头的料到完工的成品」这条真实创作链路、且在匹配质量上做得最深的拼豆工具**——一套 CIEDE2000 引擎、三种「反着用」。护城河不在「第一个做库存/实拍」（库存已有人做），而在**感知色差这层技术深度**（缺色替代、实拍容差比对、智能减色）+ **Web 端零安装的形态差异**，是国内移动 App 竞品不易顺手复制的角度。

## 市场研究发现（rev.2 附录，来源：market-pindou-differentiation-research-2026-06-30.md + App Store 截图）

- **库存驱动非空白，但有幸存差异化**：BeadMaster / Fuse Beads Studio 等已做库存盘点、图纸可拼状态、补货清单，做得很深。但均为「缺色→叫你补货」，**无人做「缺色→用手头色感知替代」**。pindou 的差异化下沉到「替代而非补货」+ Web 形态。
- **实拍纠错是唯一确证真空白**：竞品「AI 识别」最深也只是订单截图 OCR，无人做拍实物→数字图纸。乐高 Brickit 提供可行性背书。技术门槛最高、与零后端最冲突。
- **极简减色分级基本空白**：低风险、扩人群、与可达性协同；价值审美主观难验收。
- **生态层供给充足** → 支撑「避开图库/交易市场」决策。
- **跨品类铁证**：乐高 Rebrickable 把库存驱动规模化多年，证明该范式有真实刚需（但也说明不新鲜）。
- **证据缺口**：用户痛点（设计完没料/摆错返工/新手太难）因 Reddit/小红书等 UGC 社区抓取受限，**未从社区证实**，目前仍是合理假设。建议日后用真实用户访谈补强。
