# 库存驱动设计 — 体验规格

> 本规格定义库存驱动功能**如何运作**（信息架构、行为、状态、交互、无障碍、关键流程）。视觉令牌见 `./DESIGN.md`，按名引用 `{design.token}`。两份 spine 在与任何 mock 冲突时均以 spine 为准。
> **快速路径产物**：`[ASSUMPTION]` 标记的是推断，请在 review 时修正。

## Foundation

- **形态**：纯浏览器单页 Web 应用（Vue 3 + Vuetify），无登录、无后端，计算全在本地。继承既有 `StudioView` 编辑器与 `HomeView`。
- **UI 系统**：Vuetify 拼豆设计系统（见 DESIGN.md）。本 spine 只规定**行为增量**，组件视觉沿用 Vuetify 默认 + DESIGN.md。
- **新增范围**：一个独立 **Inventory（库存）顶级视图** + Studio 内一个 **库存面板**。库存数据本地持久化（localStorage/IndexedDB，介质属架构决策），架构为未来云同步预留但 v1 不做。

## Information Architecture

| Surface | Reached from | Purpose |
|---|---|---|
| Home | 应用入口 | 既有首页，不变 |
| Studio | Home / 顶栏 | 既有编辑器。**新增**：库存面板（与 PalettePanel 并列）、库存模式开关、三态灯、缺色标记 |
| **Inventory（新）** | 顶栏 / Studio 库存面板的"管理库存"入口 | 录入与维护库存：按品牌色板勾选 + 填数量。一个品牌色板一套库存 |
| Inventory · 品牌色板切换 | Inventory 内 | 在 Perler / Hama / Artkal 等之间切换，各自独立库存 |

**IA 决策（混合方案）**：录入/维护这件"重活"放独立 Inventory 页面（宽敞、不挤占编辑器）；Studio 内只放**轻量库存面板**——开关库存模式、看可拼性、触发感知替代，需要编辑库存时跳去 Inventory 页。

**Surface 闭合检查**：PRD 每个能力都有落点——库存录入 → Inventory 页（FR-1~3）；库存模式/重匹配/标记 → Studio 库存面板 + 画布（FR-4~6）；感知替代 → Studio 替代预览（FR-7~9）；可拼性/缺口清单 → Studio 库存面板（FR-10~11）。无悬空能力，也无缺少 journey 的屏。

→ 视觉参考 mock：`mockups/key-inventory.html`（Inventory 录入页）、`mockups/key-studio-panel.html`（Studio 库存面板·有缺色/全可拼对照）、`mockups/key-substitution.html`（感知替代预览卡）。**spine 在与 mock 冲突时获胜——mock 仅图示。**

## Voice and Tone

微文案。核心原则（party-mode 决议）：**引擎算得出的事，别让用户用肉眼算；说人话，不甩账本。**

| Do | Don't |
|---|---|
| 「够铺这张图，余量紧张」 | 「H2: 需求 2000 / 拥有 1900」（只甩数字） |
| 「全部可用你现有库存完成 ✓」 | 「可拼性判定：状态 GREEN」 |
| 「缺 3 个色——可以用你手头的色替代」 | 「检测到 3 个 missing color」 |
| 「用天蓝 A7 替代了浅蓝 H2（视觉影响：低）」 | 「Applied substitution: H2→A7, ΔE=1.8」 |
| 「这个色即便替代后仍差约 100 颗」 | 「Insufficient: shortfall 100」 |
| 「先录一点库存，才能开库存模式」 | 「Inventory empty. Cannot enable mode.」 |

调性沿用产品“为自己做的工具”的亲切口吻；ΔE 等技术值可作为次要信息呈现（hover/详情），但主文案一律说人话。i18n：**每条新文案必须同时补 en + zh**（构建门禁，见 DESIGN/PRD §4.5）。

## Component Patterns

行为规格。视觉见 `DESIGN.md`。

| Component | Use | 行为规则 |
|---|---|---|
| 库存条目行（Inventory 页） | 录入/维护 | 勾选框切换“是否拥有”，勾选后数量字段激活。色块 + 色号沿用 PalettePanel 风格。数量为非整数/负数时即时校验提示。未勾选项不入库存，`{design.missing-mark}` 不适用 |
| 批量操作条 | Inventory 页 | “全选/全不选”；“给已勾选项设统一默认数量”（输入一个数 → 应用到所有已勾选项，再个别微调）。降低录入成本的核心 |
| 库存模式开关 | Studio 库存面板 | `VSwitch`。库存为空时**禁用并提示**去录入（不是开了再报错）。开启后画布/材料清单切到库存色板基准。状态在会话内保持 |
| 三态灯 | Studio 库存面板 | `VChip` + `{design.buildable/substitutable/insufficient}` + 图标 + 一句人话。点击展开缺口清单 |
| 缺色格标记 | Studio 画布 | 缺色格叠加 `{design.missing-mark}` 暖橙描边 + 斜纹。数量不足色在材料清单标红，不在画布逐格标（会太吵）`[ASSUMPTION]` |
| 缺口清单 | Studio 库存面板（三态灯展开） | 每行：色块 + 色号 + **人话解读**（「够铺，余量紧张」/「还差约 X 颗」）。缺色行带“用库存替代”按钮跳 FR-7；可“复制为文本”（纯文本，**不进 PDF**） |
| 替代预览卡 | Studio（点“用库存替代”后） | 显示：将替代的缺色数、受影响格数、每个替代的视觉影响分级（低/中/高，本质 ΔE）。**前后对比**：`VBtnToggle` 切换“替代前/后”画布，或并排缩略图。底部“应用全部 / 放弃” |

## State Patterns

| State | Surface | Treatment |
|---|---|---|
| 库存为空 | Studio 库存面板 | 库存模式开关禁用；一行引导「先录一点库存，才能开库存模式」+ 跳 Inventory 按钮 |
| 库存模式关闭（默认） | Studio | 与现有基线转珠**完全一致，零回归**。无三态灯、无缺色标记 |
| 库存模式开启·全可拼 | Studio 库存面板 | 绿灯「全部可用你现有库存完成 ✓」。画布无缺色标记 |
| 库存模式开启·有缺色 | Studio | 黄灯「缺 N 个色——可以用手头的色替代」。画布缺色格暖橙标记。库存面板出现"用库存替代"主按钮 |
| 库存模式开启·数量不足 | Studio | 红灯「有 N 个色即便替代也不够」。缺口清单列出「还差约 X 颗」。不阻止用户继续——只如实告知 |
| 替代计算中 | 替代预览卡 | 短暂 loading（板尺寸级，通常瞬时，无需进度条）`[ASSUMPTION]` |
| 替代后 | Studio | 画布更新为替代结果，三态灯刷新（缺色应消失 → 转绿/黄）。可一键撤销（单条历史）回到替代前 |
| 库存色板为空但开了模式 | 替代预览 | 不产出替代，提示去录入库存 |
| 录入数量非法 | Inventory 行 | 即时内联校验：非整数/负数标红 + 一句提示，不静默吞掉 |
| 清空/重置库存 | Inventory 页 | 二次确认（`VDialog`），避免误删 |

## Interaction Primitives

- **库存模式是叠加层，不是模态**：开启不打断编辑，用户可随时画笔微调（替代只是给个好起点，不锁定）。
- **替代可逆**：应用感知替代 = 单条撤销历史，整体回退（沿用既有 useHistory 深度 50）。用户敢点，因为随时能退。
- **触发显式、不偷改**：重匹配、应用替代都是用户主动点的动作，绝不在切色板/编辑时悄悄改图（呼应 project-context “切 palette 不重匹配”）。
- **缺色 → 替代 → 可拼是一条可见的链**：黄灯 → 点替代 → 预览 → 应用 → 转绿。每一步用户都看得到因果，不是黑箱。

## Accessibility Floor

- **缺色/不足双编码**：不只靠颜色——缺色 = 暖橙 + 斜纹纹理 + 图标；不足 = 红字 + 图标（`mdi-alert`）。色盲用户可辨（DESIGN.md Do's 已钉）。
- **三态灯不只靠红黄绿**：每个状态配独立 mdi 图标（绿 = `mdi-check-circle`、黄 = `mdi-swap-horizontal`、红 = `mdi-alert-circle`）+ 文字标签，色盲与高对比模式可辨。`[ASSUMPTION]` 图标具体待定。
- **键盘可达**：库存录入（勾选、数量输入、批量）、库存模式开关、替代应用/放弃全部键盘可操作，沿用 Vuetify 默认焦点管理。
- **对比度**：沿用既有主题已验证的对比；新增的暖橙标记须在 light/dark 下都达 WCAG AA（标记叠加在彩色画布上，需描边+纹理保证可辨，不只靠色差）。
- **i18n**：所有新文案 en + zh 双语（构建门禁）。

## Key Flows

- **KF-1. 林夕用抽屉里的珠子做初音图，不再下单。**（实现 PRD UJ-1）
  - **入口**：林夕之前在 Inventory 页录过 Perler 库存（勾了二十来色、批量设了默认数量再微调）。今天进 Studio，导入初音图，转图完成。
  - **路径**：① 库存面板点开库存模式开关 → ② 画布瞬时标出缺色格（暖橙斜纹），库存面板黄灯「缺 3 个色——可以用手头的色替代」→ ③ 她点“用库存替代” → ④ 替代预览卡：「3 个缺色将用库存色替代，影响 412 格，视觉影响：低」，她切“替代前/后”对比，看那块天空没糊 → ⑤ 点“应用全部”。
  - **climax**：画布缺色标记消失，三态灯转绿「全部可用你现有库存完成 ✓」。**她不用等快递，今晚就能开摆。**
  - **resolution**：导出带色号 PDF（既有功能），按图施工。**Edge case**：若某色替代后仍数量不足，三态灯是黄灯加一行红字「天蓝 A7 替代后仍差约 100 颗」——她自己决定补这点货还是改图，工具不替她拦。

- **KF-2. 林夕开工前先问"这张图我现在能拼吗"。**（实现 PRD UJ-2）
  - **入口**：从社区存了张图纸，导入 Studio，开库存模式。
  - **路径**：库存面板三态灯直接给判定 → 点灯展开缺口清单，每行人话解读。
  - **climax + resolution**：看到黄灯「缺 2 色，可替代后无缺色」→ 放心走 KF-1 替代流程；或看到红灯「H2 还差约 1200 颗」→ 当即决定补货还是换图。**一眼定夺，不用肉眼算账。**
