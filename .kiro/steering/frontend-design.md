# 前端视觉设计规范(Vuetify)

> 改写自 Anthropic「frontend-design」skill 的设计原则,并结合本项目(Vue 3 + Vuetify 3)的技术栈落地。内容经改写以符合原始授权要求。
> 适用范围:任何新建或重做的界面。目标是做出有主见、不"一眼 AI"的界面,而不是堆默认样式。

## 核心心态

把自己当成一家精品工作室的设计主创:客户付费买的是**独特的观点**,而不是模板。每次改 UI 都要在配色、字体、布局上做出**刻意的、可被解释的选择**,并允许自己冒一个能讲清理由的审美风险。

## 先锚定主题

动手前先想清楚三件事并明确写下来:**这个界面服务的具体对象是什么、面向谁、这一屏唯一的任务是什么**。本项目是拼豆图纸设计工具(pindou-studio),用户是手工爱好者,核心动作是"上传图片 → 转成豆图 → 编辑 → 导出"。设计语言应当来自这个真实世界(像素格子、豆子颗粒感、配色卡、手作温度),而不是通用后台风。

## 设计原则

- **主视觉是一个论点(thesis)。** 首屏/主区域要开门见山地呈现这个产品最有代表性的东西(比如真实的豆图画布、转换前后对比、一个可交互的瞬间),而不是"大数字 + 小标签 + 渐变"这种模板答案。
- **字体承载个性。** 刻意搭配标题字与正文字,设定清晰的字号层级(scale)、字重与字间距。不要用在任何项目上都会随手选的那套字体。让排版本身成为记忆点,而不是中性的内容容器。
- **结构即信息。** 编号、eyebrow 小标、分隔线、标签这些结构元素应当**编码真实信息**,而不是装饰。只有当内容真的是一个有顺序的流程时,才用 01/02/03 这类编号——先质疑它是否真的成立。
- **动效要克制且有目的。** 想清楚动画服务于什么:加载序列、滚动揭示、hover 微交互、氛围感。一个被精心编排的瞬间,通常比到处散落的小特效更有力。很多时候"少即是多",过量动画反而让界面显得是 AI 生成的。
- **复杂度匹配愿景。** 极繁风格需要精致执行,极简风格需要在间距、字体、细节上的精准。优雅 = 把选定的方向执行到位。

## 避开"一眼 AI"的三种默认套路

下面三种是当前 AI 设计最常聚集的默认值,**在 brief 没有明确要求时不要用它们**:

1. 米色背景(近 #F4F1EA)+ 高对比衬线大标题 + 陶土色点缀;
2. 近黑背景 + 单一荧光绿 / 朱红点缀;
3. 报纸式排版:细线分隔、零圆角、密集多栏。

它们对某些 brief 是合理的,但当某个维度是自由的时,不要把这份自由浪费在这些默认上。

## 工作流程:头脑风暴 → 出方案 → 自我批判 → 编码 → 再批判

分两步走:

**第一步:出一个紧凑的设计方案(token 系统)**
- **Color:** 用 4–6 个命名好的 hex 值描述调色板。
- **Type:** 至少 2 个角色的字体(克制使用的特色标题字 + 互补的正文字,必要时加一个用于说明/数据的工具字)。
- **Layout:** 用一句话 + ASCII 线框图描述布局概念,方便比较多个方案。
- **Signature:** 这一屏会被记住的那个**唯一标志性元素**。

**第二步:在编码前先用 brief 审查方案。** 如果任何一部分读起来像"我对任何同类页面都会产出的通用默认值",就修改它,并说明改了什么、为什么。确认方案足够独特后,再严格按方案写代码,每个颜色和字体决策都从方案推导而来。

尽量把这些规划和迭代放在思考里完成,只有当较有把握会让用户惊喜时,才把想法展示出来。

## 克制与自我批判

- **把大胆只花在一个地方。** 让标志性元素成为唯一的记忆点,周围一切保持安静、克制,砍掉一切不服务于 brief 的装饰。(不冒险本身也是一种风险。)
- **达到质量底线但不张扬:** 响应式下探到移动端、键盘焦点可见、尊重 `prefers-reduced-motion`。
- 像香奈儿的建议:出门前照镜子,拿掉一件配饰。
- 边做边批判自己的产出,条件允许就截图看效果——一张图胜过千言。

## 文案也是设计材料

- 文字存在的唯一理由是**让界面更易理解、更易使用**。给文案和给间距、配色一样的用心。
- 站在用户这一侧写字:用人们能控制和认识的词命名,而不是系统实现的词(用户管理的是"通知",不是 webhook 配置)。
- 默认用主动语态,控件直说会发生什么:用"保存修改"而不是"提交"。一个动作在整个流程里保持同一个名字(按钮写"导出",toast 就说"已导出")。
- 把错误和空状态当作给方向的时刻,而不是渲染情绪:错误说明出了什么问题、怎么修,不道歉、不含糊;空屏是一个行动邀请。
- 语气口语、句首大写式的简洁、无废话,语调匹配品牌与受众。每个元素只做一件事。

## 本项目(Vuetify 3)落地约定

- **集中管理设计 token。** 把调色板、字体写进 Vuetify theme(`src/plugins/vuetify.ts` 的 `createVuetify({ theme })`),用 `colors`/`variables` 定义,组件里通过 `color="primary"` 等引用,**不要在组件里散落硬编码色值**。当前项目只设了 `defaultTheme: 'light'` 且无自定义 token——做视觉改版时优先在这里建立自定义主题。
- 自定义字体在 `index.html` 或全局样式引入后,通过 theme/CSS 变量统一应用,避免逐组件设置 `font-family`。
- 优先用 Vuetify 内置组件与其 props(`elevation`、`rounded`、`density`、`variant`)表达层级,而不是大量自写 CSS;确需自定义时,注意 CSS 选择器优先级,避免基于类型的选择器(如 `.section`)和基于元素的选择器互相抵消(尤其是 section 之间的 padding/margin)。
- 圆角、阴影、间距走 Vuetify 的设计尺度(spacing/elevation),保持全站一致。
- 默认提供深色主题适配能力(Vuetify theme 原生支持),除非 brief 明确只要单一主题。
- 图标统一用项目已配置的 MDI(`@mdi/font`),不混用其它图标体系。
